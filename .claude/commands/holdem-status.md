# 홀덤 프로젝트 작업 상태 확인

현재 프로젝트 작업 상태를 종합적으로 확인합니다.

## 사용법
```
/holdem-status
```

---

## 자동 실행 단계

### Step 1: 상태 파일 읽기
```
다음 파일들을 읽으세요:
1. .planning/WORK_STATE.md
2. .planning/checkpoints/ 폴더의 최신 파일 (있다면)
```

### Step 2: Git 상태 확인
```bash
git status --short
git log --oneline -3
```

### Step 3: 테스트 상태 확인 (선택적)
```bash
cd backend && pytest tests/ --collect-only -q 2>/dev/null | tail -5
```

## 상태 코드

| 상태 | 의미 |
|------|------|
| ⬜ 대기 | 아직 시작 안 함 |
| 🔄 진행중 | 현재 작업 중 |
| ⏸️ 일시중지 | 다른 작업으로 전환 |
| ✅ 완료 | 모든 Step 완료 |
| ❌ 실패 | 에러 발생, 수정 필요 |
| 🔒 블록됨 | 선행 작업 필요 |

---

## 빠른 명령어 안내

```
/holdem-fix           - 다음 작업 시작
/holdem-fix P0-1      - 특정 Phase 실행
/holdem-resume        - 중단된 작업 재개
/holdem-checkpoint    - 현재 상태 저장
```
