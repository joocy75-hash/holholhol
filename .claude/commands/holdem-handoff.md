# 홀덤 프로젝트 계정 인계

토큰 소진으로 다른 계정으로 작업을 인계할 때 사용합니다.

## 사용법
```
/holdem-handoff
```

---

## 자동 실행 단계

### Step 1: 현재 작업 즉시 저장
```
1. 수정 중인 파일 모두 저장
2. 체크포인트 자동 생성 (/holdem-checkpoint 호출)
3. WORK_STATE.md 업데이트
```

## 긴급 인계 (토큰 즉시 소진 시)

토큰이 거의 소진되어 위 과정을 완료할 수 없을 때:

### 최소 필수 액션
```
1. WORK_STATE.md의 "진행 중인 작업" 섹션만 업데이트
2. 수정 중인 파일 저장
3. 다음 메시지 출력:

"⚠️ 긴급 인계: [Phase]-[Step] 진행 중. 다음 계정에서 /holdem-resume 실행."
```

---

## 인계 받는 계정의 프로토콜

### 1. 상태 확인
```
/holdem-status
```

### 2. 인계 문서 확인
```
Read .planning/HANDOFF_*.md (가장 최신 파일)
```

### 3. 작업 재개
```
/holdem-resume
```

### 4. 작업자 등록
```
WORK_STATE.md에서 "현재 작업자" 업데이트
"계정 전환 로그"에 새 계정 기록
```
